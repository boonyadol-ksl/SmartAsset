# SmartAsset Knowledge Guide

เอกสารนี้สรุปความสามารถของแต่ละหน้าหลักในระบบ SmartAsset เพื่อใช้ประกอบการนำเสนอหรือทำความเข้าใจฟีเจอร์ของระบบ

---

## 1. `pages/dashboard.php`
- หน้าหลักภาพรวมของระบบทรัพย์สิน
- แสดงข้อมูลบริษัท/ไซต์ที่ใช้งาน
- กรองภาพรวมตามปีการตรวจนับ
- คำนวณและแสดงสถิติสำคัญ เช่น:
  - จำนวนทรัพย์สินทั้งหมด
  - จำนวนทรัพย์สินที่ใช้งานอยู่
  - จำนวนทรัพย์สินไม่ใช้งานหรือสถานะอื่น
  - มูลค่าทรัพย์สินที่ใช้งานอยู่
- แสดงกราฟและสรุปข้อมูลตาม:
  - Plant
  - สถานะทรัพย์สิน
  - แผนก
  - ประเภททรัพย์สิน
- แสดงข้อมูลทรัพย์สินล่าสุด พร้อมระบบแบ่งหน้า
- มีแท็บ Plant เพื่อสลับดูข้อมูลตาม Plant ต่าง ๆ
- รายงานความก้าวหน้าการตรวจนับจาก `audit_assignments`

## 2. `pages/asset-list.php`
- หน้ารายการทรัพย์สินทั้งหมด
- รองรับการค้นหาและกรองข้อมูลด้วย:
  - คำค้นหา (Asset No, รายละเอียด, Serial No, ผู้ดูแล)
  - Plant
  - Cost Center
  - แผนก
  - สถานะ
  - municipality
- รองรับการปรับจำนวนรายการต่อหน้า และ pagination
- แสดงปุ่ม:
  - นำเข้า Excel (เฉพาะ admin)
  - เพิ่มทรัพย์สินใหม่ (เฉพาะ admin)
- รองรับ Export CSV ของข้อมูลที่กรองแล้ว
- แสดงสถานะทรัพย์สินด้วย Badge สีต่าง ๆ
- มีปุ่มดำเนินการสำหรับแต่ละรายการ: ดู, แก้ไข, ลบ (admin)

## 3. `pages/asset-detail.php`
- หน้ารายละเอียดทรัพย์สิน
- ทำได้ทั้ง:
  - ดูข้อมูลทรัพย์สิน
  - แก้ไขข้อมูล
  - เพิ่มทรัพย์สินใหม่
- ฟอร์มข้อมูลทรัพย์สินประกอบด้วย:
  - Plant
  - Asset No
  - ประเภททรัพย์สิน
  - วันที่ซื้อ
  - รายละเอียด
  - ยี่ห้อ/รุ่น
  - Serial No
  - มูลค่าซื้อ
  - Cost Center
  - แผนก
  - ผู้ดูแล
  - สถานะ
- โหลด dropdown ประเภททรัพย์สิน, แผนก, Plant
- แสดงภาพทรัพย์สิน, QR code, และประวัติการตรวจนับย้อนหลัง
- แสดงข้อมูล audit/inventory history รายการล่าสุด
- มีปุ่มพิมพ์ทรัพย์สินเมื่อต้องการสร้างเอกสาร

## 4. `pages/asset-import.php`
- หน้านำเข้าทรัพย์สินทางไฟล์จำนวนมาก
- รองรับการอัปโหลดไฟล์ `xlsx`, `xls`, `csv`
- มีโหมดการจัดการกรณี Asset No ซ้ำ:
  - update: อัปเดตทับ
  - skip: ข้ามรายการซ้ำ
  - new_only: นำเข้าเฉพาะรายการใหม่
- แสดงโปรเซสการอัปโหลดและสรุปผล
- แสดงประวัติการนำเข้า (import logs)
- มีลิงก์ดาวน์โหลด Template
- แสดงคู่มือคอลัมน์ที่รองรับสำหรับการนำเข้า
- หน้านี้ใช้เฉพาะผู้มีบทบาท `admin`

## 5. `pages/asset-print.php`
- หน้าเอกสารพิมพ์ทรัพย์สินแบบ A4
- รองรับการส่ง `id` หลายรายการเพื่อพิมพ์ทีละหลายแผ่น
- แสดงข้อมูลทรัพย์สินพร้อม audit ล่าสุด
- แสดงรูปทรัพย์สิน, รูป QR code, และรูปจากการตรวจนับ
- คำนวนอายุทรัพย์สินจากวันที่ซื้อ
- สร้างรายงานในรูปแบบที่เหมาะกับการพิมพ์

## 6. `pages/inventory.php`
- หน้าตรวจนับทรัพย์สิน / Audit workflow
- รองรับ AJAX endpoint หลัก เช่น:
  - `get_signature` / `save_signature`
  - `get_cost_centers`
  - `search_by_qr`
  - `get_audit`
  - `save_audit`
- สนับสนุนการค้นหาผ่าน QR code หรือ Asset No
- จัดการสถานะการตรวจนับของ assignment
- บันทึกผลการตรวจนับพร้อม:
  - สถานะตรวจพบ/ไม่พบ/ชำรุด/ไม่ใช้งาน/รอซ่อม
  - จำนวน
  - สถานที่
  - ผู้รับผิดชอบ
  - หมายเหตุ
  - รูปถ่ายและลายเซ็น
- จำกัดการเข้าถึงตามบทบาทผู้ใช้ เช่น inventory, webadmin, admin

## 7. `pages/inventoryx.php`
- เวอร์ชันอีกหน้าหนึ่งของระบบตรวจนับทรัพย์สิน
- มีฟังก์ชันใกล้เคียงกับ `inventory.php`
- สนับสนุน QR search, get_audit, save_audit, get_cost_centers
- ใช้โครงสร้างหน้าและ AJAX workflow ที่คล้ายกัน
- ดูเหมือนเป็นหน้าสำหรับใช้จัดการ audit assignment และบันทึกผลตรวจนับในรูปแบบ UI ที่อาจใหม่กว่าหรือแตกต่าง

## 8. `pages/manage_audit.php`
- หน้าสร้างรอบการตรวจนับใหม่
- รองรับการกำหนด:
  - Plant
  - ปีการตรวจนับ
  - เป้าหมาย % การตรวจ
  - วันสิ้นสุดรอบ (Deadline)
- เลือก Cost Center เพื่อกำหนดขอบเขตงาน
- แสดงสถานะของ Cost Center ว่า:
  - ยังมีรายการคงเหลือ
  - ตรวจครบแล้ว
  - มอบหมายบางส่วนแล้ว
- มอบหมายงานให้ผู้ตรวจนับ (inspectors)
- แสดง workload ปัจจุบันของผู้ตรวจนับ
- มีฟังก์ชันยกเลิกงาน pending ที่มอบหมายแล้ว
- แสดงผลลัพธ์หลังสร้างการมอบหมายงาน

## 9. `pages/reports.php`
- หน้ารายงานทรัพย์สินและการตรวจนับ
- รองรับการเลือก:
  - ปีการตรวจนับ
  - ช่วงย้อนหลังหลายปี
  - Plant
- คำนวนข้อมูลจาก `audit_assignments`, `audit_sessions`, `assets`
- แสดงสรุปเช่น:
  - จำนวนทรัพย์สินที่ตรวจแล้ว/ยังว่าง
  - สถานะการตรวจนับตามแผนก
  - มูลค่าทรัพย์สินที่ตรวจแล้ว
- มีรายงานแยกตาม:
  - แผนก
  - ประเภททรัพย์สิน
  - แผนก
  - กราฟการซื้อสินค้าตามเดือน
- มีปุ่ม Export CSV สำหรับ:
  - Assets
  - Dept report
  - Class report
  - Monthly report
  - All summary
- มีปุ่มพิมพ์รายงาน

## 10. `pages/report-print.php`
- หน้าพิมพ์รายงานแบบเรียบร้อย
- รองรับรูปแบบรายงาน:
  - `summary` รายงานสรุปการตรวจนับ
  - `checklist` ใบตรวจนับทรัพย์สิน
  - รายงานรายละเอียดทรัพย์สิน
- กรองข้อมูลตาม Plant / Department / Session
- แสดงตารางสรุปและข้อมูลตรวจนับ
- รวมบรรทัดสำหรับลายเซ็นด้วย

## 11. `pages/password.php`
- หน้าเปลี่ยนรหัสผ่านของผู้ใช้งาน
- ฟอร์มรับ:
  - รหัสผ่านปัจจุบัน
  - รหัสผ่านใหม่
  - ยืนยันรหัสผ่านใหม่
- ตรวจสอบความยาวขั้นต่ำและการยืนยันรหัสผ่าน
- ส่งข้อมูลไปยัง API `api/users.php?action=change_password`

## 12. `pages/sites.php`
- หน้าจัดการ Site สำหรับผู้ใช้งาน `admin` และ `webadmin`
- ฟีเจอร์หลัก:
  - แสดงรายการ Site
  - เพิ่ม Site ใหม่
  - แก้ไขข้อมูล Site
  - เปิด/ปิดการใช้งาน Site
  - ผูก Plant เข้ากับ Site
- มีข้อจำกัดการเข้าถึงตามบทบาทผู้ใช้
- ใช้ API `api/sites.php` ในการดึง/บันทึกข้อมูล

## 13. `pages/user.php`
- หน้าจัดการผู้ใช้งานระบบ
- สำหรับบทบาท `admin` และ `webadmin`
- ฟีเจอร์หลัก:
  - แสดงตารางผู้ใช้ทั้งหมด
  - เพิ่มผู้ใช้ใหม่
  - แก้ไขข้อมูลผู้ใช้
  - ลบผู้ใช้ (ไม่สามารถลบตัวเอง)
  - กำหนดสิทธิ์ `admin`, `webadmin`, `inventory`, `viewer`
  - จัดการ Site ของผู้ใช้งาน (ยกเว้น admin)
  - กำหนดสถานะ Active / Inactive
- มี modal สำหรับสร้างและแก้ไขผู้ใช้
- ใช้ API `api/users.php` ในการดึงและบันทึกข้อมูล

---

## หมายเหตุเพิ่มเติม
- ระบบใช้ `components/head.php` และ `components/sidebar.php` เป็นโครงสร้างหน้า common
- ไฟล์ JavaScript ร่วมอยู่ใน `assets/js/app.js`
- บทบาทที่สำคัญ:
  - `admin` = ควบคุมสูงสุด, จัดการผู้ใช้, Site, นำเข้า, แก้ไขทรัพย์สิน
  - `webadmin` = จัดการไซต์ของตัวเอง / ดูแลการมอบหมาย
  - `inventory` = เข้าถึงฟังก์ชันตรวจนับ
  - `viewer` = อ่านข้อมูลได้เป็นหลัก

---

## ข้อเสนอแนะการนำเสนอ
- เริ่มด้วยภาพรวม `Dashboard` เป็นจุดเข้าใช้งานหลัก
- แยกเป็น 3 กลุ่มฟีเจอร์หลัก:
  1. จัดการทรัพย์สิน (`asset-list`, `asset-detail`, `asset-import`, `asset-print`)
  2. ตรวจนับและมอบหมายงาน (`inventory`, `inventoryx`, `manage_audit`)
  3. รายงานและพิมพ์สรุป (`reports`, `report-print`)
- ปิดท้ายด้วยหัวข้อระบบผู้ใช้งานและการตั้งค่า (`password`, `sites`, `user`)
