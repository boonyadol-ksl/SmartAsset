คุณครับ ว่า ในเมนูตรวจนับ จำเป็นต้องมีอะไรบ้าง

- สรุปรายการตรวจนับตารางเห็นชัดเจน
- กรอง ข้อมูลได้ทุกอย่าง เช่น plant -> cost_center -> ผู้ตรวจนับ รหันทรัพย์สิน
- สามารถ สแกน QRcode ได้ ถ้าเจอ 1 รายการก้แสดง modal ขึ้นมาเลย ถ้าเยอะก้แสดงรายการ
- สามารถ update ข้อมูลทรัพย์สินได้ จำนวน รูปภาพทรัพย์สิน รูปqrcode จริง หมายเหตุ สถานะ ลายเซ็น location
  แนวคิดการออกแบบเมนูตรวจนับ (Inventory/Audit Menu) ที่คุณลิสต์มา ถือว่าครบถ้วนและตอบโจทย์การทำงานจริงของ User หน้างาน (เช่น เจ้าหน้าที่สแกน) และผู้บริหารระบบ (Admin) มาก ๆ ครับเพื่อให้เห็นภาพการนำไปพ่นโค้ดหรือทำหน้า UI ชัดเจนขึ้น ผมขอสรุปและขยายความความจำเป็นในแต่ละส่วน พร้อมแนะนำฟิลด์ข้อมูลที่ต้องเชื่อมโยงจาก Database ให้เห็นภาพดังนี้ครับ:📊 1. ส่วนสรุปรายการตรวจนับ (Dashboard / Summary Table)ส่วนนี้จำเป็นต้องแบ่งเป็น 2 มิติ เพื่อให้ส่องดูข้อมูลได้ง่าย:ส่วนบน (Overview Cards): แสดงตัวเลขสรุปไว ๆ เช่นจำนวนที่ต้องตรวจทั้งหมด (Total)ตรวจแล้ว (Completed)คงเหลือ (Pending)เปอร์เซ็นต์ความคืบหน้า (Progress Bar)ส่วนตาราง (Main Data Table): แสดงรายการละเอียด โดยดึงโครงสร้างจากตารางของคุณมาโชว์:รหัสทรัพย์สิน (asset_no) | ชื่อทรัพย์สิน (asset_description) | โรงงาน (plant_code) | หน่วยงาน (cost_center)ผู้ตรวจนับ (full_name)สถานะล่าสุด (check_status เช่น found, not_found, damaged)วันเวลาที่ตรวจล่าสุด (checked_at)🔍 2. ระบบกรองข้อมูล (Multi-Level Filters)ควรทำระบบกรองแบบ Cascading Dropdown (เลือกข้อบนแล้วข้อล่างเปลี่ยนตาม) เพื่อไม่ให้สับสน:Level 1: เลือกปี หรือ รอบการตรวจ (audit_sessions.id)Level 2: เลือกโรงงาน (plants.plant_code หรือ assets.plant_code)Level 3: เลือกหน่วยงาน (assets.cost_center) จะโชว์เฉพาะที่มีใน Plant นั้นLevel 4: เลือกผู้ตรวจนับ (users.id)ช่องค้นหาอิสระ (Global Search): ให้พิมพ์เพื่อค้นหา รหัสทรัพย์สิน (asset_no), Serial No (serial_no), หรือชื่อทรัพย์สินได้ทันที📷 3. ระบบสแกน QR Code (Smart Scanner Logic)ตรรกะ (Logic) ที่คุณวางไว้ดีมากครับ ช่วยลดเวลาหน้างานได้เยอะ:เปิดกล้องเว็บ/มือถือ: สแกนค่าจาก QR Code (assets.qr_code)Logic การค้นหา:🟢 กรณีเจอ 1 รายการตรง ๆ: ระบบจะดึงข้อมูลฟิลด์ขึ้นมา และเด้งหน้าต่าง Modal สำหรับอัปเดตข้อมูลการตรวจนับขึ้นมาทันที (ไม่ต้องให้ User กดเลือกซ้ำซ้อน)🟡 กรณีเจอหลายรายการ (เช่น QR code เดียวพ่วงสินทรัพย์ย่อยหลายตัว): ให้แสดงเป็นรายการ List ในหน้าจอหลัก เพื่อให้ User กดเลือกตัวที่ถูกต้องก่อน แล้วค่อยเปิด Modal🔴 กรณีไม่พบข้อมูล: แสดงข้อความเตือน (Alert) ว่า "ไม่พบรหัสทรัพย์สินนี้ในระบบรอบการตรวจปัจจุบัน"📝 4. หน้าต่างอัปเดตข้อมูลตรวจนับ (Update Modal / Form)เมื่อสแกนเจอหรือกดคลิกตรวจนับ ข้อมูลฟิลด์ที่คุณระบุมาจะถูกบันทึกลงตาราง inventory_results และบางส่วนจะเชื่อมไปอัปเดตที่ assets ครับ โดยควรออกแบบฟอร์มดังนี้:สิ่งที่ต้องการอัปเดตฟิลด์ใน Database ที่รองรับรูปแบบ UI ที่แนะนำบนหน้าจอจำนวนinventory_results.quantityช่องใส่ตัวเลข (Number Input) มีปุ่ม + / -รูปภาพทรัพย์สินinventory_results.photo_pathปุ่มกดเปิดกล้องถ่ายภาพจริงหน้างาน หรืออัปโหลดไฟล์รูป QR Code จริงสามารถเก็บพ่วงใน photo_path หรือทำฟิลด์เพิ่มปุ่มถ่ายภาพสภาพสติกเกอร์ QR Code ที่ติดอยู่กับตู้/เครื่องจักรหมายเหตุinventory_results.remarksช่องพิมพ์ข้อความขยายใหญ่ (Textarea)สถานะinventory_results.check_statusปุ่มตัวเลือก (Radio หรือ Dropdown): found (เจอ), not_found (ไม่เจอ), damaged (ชำรุด)ลายเซ็นinventory_results.signatureช่องแคนวาส (Signature Pad) ให้ User ใช้นิ้วเซ็นสดบนหน้าจอได้เลยLocationinventory_results.locationช่องพิมพ์ หรือปุ่มดึงพิกัด GPS/ระบุชื่อห้องที่ตรวจพบ
